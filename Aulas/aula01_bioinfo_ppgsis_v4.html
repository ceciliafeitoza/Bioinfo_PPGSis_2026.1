O que é Bioinformática?

Bioinformática é a aplicação de métodos computacionais e estatísticos para organizar, analisar e interpretar dados biológicos — especialmente dados moleculares em larga escala.

Três pilares:

  • Biologia — a pergunta e o contexto
  • Computação — as ferramentas e algoritmos
  • Estatística — a interpretação dos resultados
Onde a Bioinformática atua
🧬 Genômica
🦠 Metagenômica
🌿 Filogenômica
📊 Pop. Genética
🔬 Transcriptômica
💊 Biologia Estrutural

Bioinformática não é só “rodar programas” — é integrar biologia, computação e estatística para responder perguntas biológicas. A pergunta sempre vem da biologia; as ferramentas são computacionais; a interpretação exige estatística. Sem os três pilares, a análise fica incompleta.

GUI vs CLI — duas formas de operar um computador

🖱️ GUI
Graphical User Interface
  • Cliques, janelas, menus
  • Intuitivo para iniciantes
  • Difícil de automatizar
  • Não escala para grandes volumes
  • Exemplos: Windows Explorer, RStudio, Geneious
⌨️ CLI
Command Line Interface
  • Comandos de texto no terminal
  • Curva de aprendizado inicial
  • Totalmente automatizável
  • Escala para milhares de amostras
  • Exemplos: Bash, Terminal, SSH

Tip

Em bioinformática: quase todas as ferramentas são CLI. Dominar o terminal não é opcional.

A maioria das pessoas passa a vida inteira usando GUI — clicando em ícones e menus. Em bioinformática, isso não funciona. As ferramentas são CLI porque precisam rodar em servidores remotos, processar centenas de amostras em paralelo, e ser automatizadas em pipelines. Hoje vocês darão os primeiros passos no terminal.

Boas práticas em bioinformática

📁 Organização

Estrutura consistente desde o início:

projeto/
├── data/raw/
├── data/processed/
├── scripts/
├── results/
└── docs/

🔄 Reprodutibilidade

Tudo deve poder ser refeito:

  • Scripts comentados
  • Versões de ferramentas registradas
  • Ambiente documentado

📝 Documentação

Registre o que e por quê:

  • Notebooks Quarto
  • README em cada pasta
  • Commits descritivos

Três princípios que guiam toda a disciplina. Organização: projeto bagunçado é projeto morto. Reprodutibilidade: se você não consegue replicar sua análise, ela não é ciência. Documentação: o maior beneficiário é o você do futuro.

  2. Clique em Fork (canto superior direito)

Mais tarde você aprenderá a sincronizar seu fork via Pull Request — é como colaboração em código aberto funciona.

Git & GitHub · Controle de Versão

Nunca mais analise_final_v3_DEFINITIVO_2.R

Por que controle de versão?

Sem Git:

analise.R
analise_v2.R
analise_final.R
analise_DEFINITIVO.R
analise_USAR_ESSE.R

Qual é o atual? O que mudou entre versões?

Com Git:

analise.R  ← apenas um arquivo
git log --oneline
a3f2c1d Corrige filtro de qualidade
b8e4d2a Adiciona suporte paired-end
c1a9f3e Versão inicial do pipeline

Histórico completo, sem duplicatas.

Quem já teve uma pasta parecendo com a da esquerda? Git resolve isso completamente.

O fluxo do Git

📝
Working Directory
seus arquivos locais
arquivos modificados aparecem em vermelho no git status
git add
→
📦
Staging Area
selecionados para o commit
arquivos prontos aparecem em verde no git status
git commit
→
🏠
Repositório Local
histórico no seu computador
snapshot permanente com mensagem
git push
→
☁️
GitHub (Remoto)
backup + colaboração
visível para colaboradores
git status → git add . → git commit -m "mensagem" → git push origin main

Esse fluxo é o coração do Git. Working directory é onde você trabalha. Staging é a área de preparação. Commit é o snapshot. Push envia para o GitHub. Importante: commit sem push só existe localmente.

Os 4 comandos essenciais

git status                              # ver o que mudou
git add nome_do_arquivo.R               # selecionar para o commit
git commit -m "Descrição da mudança"    # registrar snapshot
git push origin main                    # enviar para o GitHub

Para receber atualizações do repositório original:

git pull origin main                    # baixar mudanças remotas

Git tem dezenas de comandos, mas no dia a dia você usa principalmente quatro — e o pull quando trabalha em equipe.

Exercício — primeiro commit

# Criar arquivo de notas
echo "# Notas - Aula 1 Bioinformática" > notas_aula1.md
echo "Data: 15/06/2026" >> notas_aula1.md

# Ver o status
git status

# Adicionar, commitar e publicar
git add notas_aula1.md
git commit -m "Adiciona notas da primeira aula"
git push origin main

Important

Verifique no GitHub — o arquivo deve aparecer no seu repositório.
Esse é o objetivo antes de sair hoje.

Este é o exercício mais importante do dia. Quero que todo mundo faça um commit e push antes de sair.

Boas mensagens de commit

Evite:

git commit -m "correção"
git commit -m "mudanças"
git commit -m "versão final"
git commit -m "."

Prefira:

git commit -m "Corrige filtro de reads curtas"
git commit -m "Adiciona suporte a paired-end"
git commit -m "Atualiza parâmetros do Trimmomatic"

Tip

Regra: complete a frase “Este commit ______”

Mensagens de commit são um hábito pequeno com grande impacto.
